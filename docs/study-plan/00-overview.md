# AI 爆款文章创作器 — 从零到一学习计划

## 项目总览

你要从零实现一个**多智能体协作的智能图文创作平台**。用户输入一个主题，5 个 AI Agent 分工协作，经历"选题 → 大纲 → 正文 → 配图 → 合成"三阶段流程，每阶段可由用户介入调整，最终输出一篇图文并茂的文章。

核心难点不在 CRUD，而在：**多智能体编排 + SSE 流式推送 + 配图策略模式**。抓住这三条主线，项目就能立起来。

## 模块拆分与依赖关系

```
模块1: 基础架构搭建
  │
  ├─→ 模块2: 用户模块
  │     │
  │     └─→ 模块3: 文章基础模块
  │           │
  │           ├─→ 模块4: SSE 实时通信
  │           │     │
  │           │     └─→ 模块5: 智能体模块（核心）
  │           │           │
  │           │           └─→ 模块6: 配图服务模块
  │           │
  │           └─→ 模块7: 支付与会员模块
  │
  └─→ 模块8: 前端应用（可与后端模块 3-6 并行推进）
```

## 每日安排（10 天）

| 天数 | 模块 | 交付目标 |
|------|------|---------|
| Day 1 | 模块1 基础架构 | 项目能启动，连上 MySQL + Redis，健康检查接口返回 OK |
| Day 2 | 模块2 用户模块 | 注册/登录/登出跑通，Session 写入 Redis，管理员接口有权限拦截 |
| Day 3 | 模块3 文章基础 | 文章 CRUD + 状态机，能创建文章、查列表、看详情、删除 |
| Day 4 | 模块4 SSE 通信 | SSE 连接建立，能向前端推送消息，前端 EventSource 能接收 |
| Day 5 | 模块5 智能体（上） | 单 Agent 跑通：标题生成 Agent 调通 DashScope LLM，结果落库 |
| Day 6 | 模块5 智能体（下） | 三阶段编排跑通：大纲流式输出 → 正文流式输出 → 配图分析 → 合成 |
| Day 7 | 模块6 配图服务 | Pexels 配图跑通 + 策略模式框架，至少实现 2 种配图方式 |
| Day 8 | 模块7 支付会员 | Stripe Checkout 跑通 + 配额扣减 + VIP 权限校验 |
| Day 9 | 模块8 前端（上） | 登录页 + 文章创建页（三阶段交互 + SSE 实时展示） |
| Day 10 | 模块8 前端（下） | 文章列表/详情 + 管理后台 + 整体联调 |

## 学习策略

1. **先跑通再优化**：每个模块先实现最小可用版本，不要一开始就追求完美架构
2. **参考源码但不抄**：每个设计文档会给出设计意图和关键接口定义，具体实现自行完成，遇到卡点再对照源码
3. **逐模块验证**：每完成一个模块，确保接口可调通再进入下一个
4. **前端后端交替**：Day 4 之后就可以边写后端边写前端页面，保持成就感
